NAME

b58-dec - decode a base 58-encoded string  

SYNOPSIS

[-f ] [-d ] [-x ] [B58STRING]  

DESCRIPTION

decodes its input from base58 and writes the result to standard output, as a byte string by default. If B58STRING is given, the input is taken from that argument. Otherwise the input is taken from standard input, in which case it's allowed to contain a single newline at the end.

The default base58 encoding is the Bitcoin encoding. See b58(1) for a fuller discussion of the available encodings.

For output formats that are strings representing numbers (see -d and -h), and if the Bitcoin encoding scheme is in use, then any leading null bytes that would exist in the byte string are reported to standard error in the form "null bytes: N", followed by a newline, where N is the number of leading null bytes. In this case, unless fails, this will be the only line written to standard error.

Options are:

-f
Flickr-style encoding.
-d , -decimal
Output the decoded value as a string representing a decimal number, followed by a newline. Leading zero bytes for the Bitcoin scheme are represented as described above.
-x , -hex
Output the decoded value as a string representing a hexadecimal number, including a leading "0x", followed by a newline. Leading zero bytes for the Bitcoin scheme are represented as described above.
 

EXIT STATUS

Ex -std b58

A usage error, including providing any invalid data to decode, results in an exit code of 64.  

SECURITY CONSIDERATIONS

On most systems, argument lists are visible to other users. So if you're processing confidential data, feed its input on its standard input, rather than in B58STRING
